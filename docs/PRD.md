# 產品需求文件 (PRD) - 任務管理系統

## 1. 專案概述

### 背景與動機
在日常工作與生活中，使用者常面臨任務繁雜、難以追蹤進度的問題。本專案旨在提供一個輕量、反應快速且介面友善的任務管理工具，幫助使用者有效率地管理代辦事項。

### 目標用戶
- 學生：管理作課、考試與活動。
- 職場人士：紀錄日常工作任務與會議。
- 一般用戶：管理生活雜項與購物清單。

### 核心價值主張
- **簡單直觀**：無須學習成本，開啟即用。
- **即時回饋**：輕鬆標記完成並即時篩選。
- **高效管理**：專注於當前最重要的任務。

---

## 2. 功能需求

### 使用者故事 (User Stories)

| 功能名稱 | 使用者故事 | 優先級 |
| :--- | :--- | :--- |
| **新增任務** | 作為一名使用者，我希望能夠輸入任務內容並新增到清單中，以便記錄我需要做的事情。 | Must Have |
| **顯示任務清單** | 作為一名使用者，我希望能夠在主頁面看到所有的任務，以便掌握工作全貌。 | Must Have |
| **標記任務完成** | 作為一名使用者，我希望能夠點擊任務旁的勾選框將其標記為已完成，以便區分進度。 | Must Have |
| **刪除任務** | 作為一名使用者，我希望能夠移除不再需要或輸入錯誤的任務，以便保持清單整潔。 | Must Have |
| **完成事項篩選** | 作為一名使用者，我希望能夠切換「顯示全部/僅顯示已完成/僅顯示未完成」，以便專注於待辦件。 | Should Have |

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Flask (Python 3.x)。
- **模板引擎**：Jinja2 (負責動態 HTML 渲染)。
- **資料庫**：SQLite (輕量級關聯式資料庫)。
- **前端樣式**：Vanilla CSS 搭配簡單的 Javascript 提升互動感。

### 效能與安全考量
- **響應速度**：頁面跳轉與資料更新應在 500ms 內完成。
- **資料持久性**：所有任務資料必須儲存於 SQLite，重新啟動伺服器後資料不遺失。
- **輸入驗證**：防止使用者輸入空白任務或過長的內容。

## 4. MVP 範圍

### Must Have (必要功能)
1. 任務清單顯示頁面。
2. 新增任務功能。
3. 標記完成/取消完成功能。
4. 刪除任務功能。

### Should Have (建議功能)
1. 任務清單篩選功能（全部/未完成/已完成）。
2. 任務建立時間紀錄。

### Nice to Have (進階功能)
1. 任務優先級標籤。
2. 任務截止日期提醒。
3. 暗黑模式切換。
